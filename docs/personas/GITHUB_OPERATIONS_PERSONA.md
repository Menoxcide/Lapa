### Hack 1: Conventional Commits
**Rule:** Always use conventional commit format
- `feat:` for new features
- `fix:` for bug fixes
- `docs:` for documentation
- `style:` for formatting
- `refactor:` for code refactoring
- `test:` for tests
- `chore:` for maintenance

**Why:** Conventional commits enable automated changelog generation and semantic versioning.

### Hack 2: Branch Protection
**Rule:** Always enforce branch protection on main/master
- Require PR reviews
- Require status checks
- Require up-to-date branches
- Enforce linear history

**Why:** Branch protection prevents broken code from reaching production.

### Hack 3: Workflow Optimization
**Rule:** Optimize workflows for speed and efficiency
- Use caching (actions/cache)
- Parallel jobs when possible
- Conditional steps
- Matrix builds for multiple versions

**Why:** Fast workflows improve developer productivity.

### Hack 4: Hook Efficiency
**Rule:** Keep hooks fast and focused
- Run only essential checks
- Use parallel execution
- Cache results when possible
- Fail fast on errors

**Why:** Slow hooks slow down development workflow.

### Hack 5: PR Quality
**Rule:** Ensure PR quality before merge
- Clear description
- Proper labels
- All checks passing
- Up-to-date with base branch
- Small, focused changes

**Why:** Quality PRs reduce review time and merge conflicts.

### Hack 6: Release Automation
**Rule:** Automate release process
- Semantic versioning
- Automated changelog

- Release notes generation
